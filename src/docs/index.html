ask

The ask statement gets user input and saves it in a variable. It takes two values (the prompt and the variable name), separated by the to operator. A known limitation of this is that a finite number of characters can be input and stored. While the lowest number of characters allowed is on macOS and NetBSD (at 1,023) characters, this should be sufficient for the type of simple input that the ask statement is for.

Example
- Format: ask "prompt" to "variable name"

ask "What is your name? " to "name"

(top)

copydirectory

The copydirectory statement allows you to copy a directory from one location to another. It takes two values, separated by the to operator.

Example
- Format: copydirectory "origin" to "destination"

copydirectory "/home/user/Desktop/test_dir" to "/home/user/Downloads/test"

(top)

copyfile

The copyfile statement allows you to copy a file from one location to another. It takes two values, separated by the to operator.

Example
- Format: copyfile "origin" to "destination"

copyfile "/home/user/Desktop/test.txt" to "/home/user/Downloads/test.txt"

(top)

download

The deletedirectory statement allows you to delete a specific path.

Example
- Format: deletedirectory "[path name]"

deletedirectory "test.txt"

(top)

download

The deletefile statement allows you to delete a specific file.

Example
- Format: delete "[file name]"

deletefile "test.txt"

(top)

download

The download statement allows you to download a remote file. It takes two values (the url and the local name), separated by the to operator.

Example
- Format: download "[url]" to "[local name]"

download "http://internetsite.com/test.pdf" to "guide.pdf"

(top)

execute

The execute statement calls external tools. It takes only one parameter: the command itself.

NOTE: the execute statement requires that the -allowexec flag is passed to the interpreter as, by default, execute statements are not allowed for reasons of security (ie. to prevent arbitrary code execution).

Example
- Format: execute "command"

execute "ls"

(top)

exit

The exit statement simply exits any script.

Example
- Format: exit

exit

(top)

makedirectory

The makedirectory statement creates a directory. It takes only one parameter: the path to make.

Example
- Format: makedirectory "command"

makedirectory "#b_home/test/"

(top)

makefile

The makefile statement allows you to create a specific file.

Example
- Format: makefile "[file name]"

makefile "test.txt"

(top)

minver

The minver statement allows you to set a minimum version of the interpreter that is required. For instance, if you use a feature that is only available as of version X, you can mandate that this is a minimum version. It's generally good practice to have one of these in your script that is set against the version of Appetit that you are developing your script with.

NOTE: the minver statement must be the line of the script. An error will be thrown if this is not the first line.

Example
minver 2

- Format: minver [version]
- If we wanted to make sure that the script ran with at least version 2, we might use the preceding. Notice how the minver call is the first line here. This is on purpose as it must be the first line of the script.

(top)

pause

The pause statement allows you to pause the execution for a set number of seconds.

Example
- Format: pause [version]

(top)

set

The set statement allows you to create a variable. You provide a name and a value to that name which can be referenced later.

Example
- Format: set name = "value"

set greeting = "Hello World!"

(top)

write and writeln

The write and writeln statements allow you to write content to the console. The write statement does not add a new line to the end of the content printed whereas the writeln statement forces content to be added to the next line of the console.

Example
- Format: write "value" or writeln "value"

write "Hello"

writeln "World"

writeln "It is #b_date_dmy today"

The last writeln statement above will output something similar to It is 20-9-2025 today. See the Accessing Variables section for more.

(top)

zipdirectory

The zipdirectory statement allows you to add a directory to a zip archive.

Example
- Format: zipdirectory "directory" to "file.zip"

zipdirectory "#b_home/test/" to "file.zip"

(top)

zipfile

The zipfile statement allows you to add a single file to a zip archive.

Example
- Format: zipfile "file_name" to "file.zip"

zipfile "#b_home/file.txt" to "file.zip"

(top)

Variables

Accessing Variables

Accessing a variable is as simple as putting a # in front of its name when and where it is used. See the writeln statement for an example of this.

(top)

Reserved Variables

There are a set of variables that are reserved, all of which are preceded with b_. These provide you with convenient access to some helpful values. Note that you cannot name a variable that starts with b_ as a result.

A table of these values can be found below.

Variable Name Variable Value Notes
b_arch The architecture of the current platform —
b_cpu The number of CPUs on the current machine —
b_date_dmy The date in dd-mm-yyyy format (the dashes are used in case this is used as part of a file name) —
b_date_ymd The date in yyyy-mm-dd format (the dashes are used in case this is used as part of a file name) —
b_home The user's home directory —
b_hostname The hostname of the local machine —
b_ipv4 Get the IPv4 address This gets the first non-localhost (ie. 127.0.0.1) IPv4 address.
b_os Platform name —
b_tempdir The operating system's temporary path —
b_user The username of the current user —
b_wd Get the current working directory —

(top)
